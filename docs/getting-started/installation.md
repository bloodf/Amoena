# Installation

## Prerequisites

Before building Amoena from source, ensure you have the following installed:

### Required

| Tool        | Version | Install                            |
| ----------- | ------- | ---------------------------------- |
| **Node.js** | 22+     | [nodejs.org](https://nodejs.org)   |
| **Bun**     | 1.1+    | [bun.sh](https://bun.sh)           |
| **Git**     | Latest  | [git-scm.com](https://git-scm.com) |

### Platform Notes

Amoena ships as an Electron desktop app. You do not need Rust or the previous native desktop toolchain.

- **macOS**: Xcode Command Line Tools are recommended for native module builds and signing workflows.
- **Windows**: Visual Studio Build Tools may be required for native dependencies.
- **Linux**: Install the usual desktop runtime libraries for your distro if Electron reports a missing system package.

## Clone and Install

```bash
# Clone the repository
git clone https://github.com/AmoenaAi/amoena.git
cd amoena

# Install workspace dependencies
bun install
```

## Running in Development

```bash
# Start the Electron desktop app in development mode
bun run desktop:dev

# Optional: run the mobile companion in Expo
bun run mobile:dev
```

This starts the Electron shell and local runtime needed by the desktop app. Paired mobile clients can then connect through the remote access flow.

If you want to work inside the desktop workspace directly, use `bun run --cwd apps/desktop dev`.

## Building from Source

```bash
# Build the desktop app bundles used by the Electron shell
bun run desktop:build

# Package platform installers
bun run --cwd apps/desktop electron:build
```

Packaged artifacts are produced by `electron-builder` from `apps/desktop/`.

| Platform | Output                                           |
| -------- | ------------------------------------------------ |
| macOS    | `.app` + `.dmg`                                  |
| Linux    | `.AppImage` and other configured Linux targets   |
| Windows  | installer output generated by `electron-builder` |

## Verifying the Build

After installation, launch Amoena. The setup flow walks through:

1. Selecting your default AI provider
2. Entering your API key
3. Configuring your default model
4. Optional: enabling mobile pairing and remote access

See [Configuration](/getting-started/configuration) for detailed settings documentation.

## Troubleshooting

### `bun install` fails

Make sure Bun and Node are on supported versions:

```bash
bun --version
node --version
```

### Desktop dev server does not start

```bash
bun run desktop:dev
```

If Electron reports a missing dependency, install the required system package for your platform and rerun the command.

### Packaging fails

Run the packaging command from the desktop workspace to get the full Electron builder output:

```bash
bun run --cwd apps/desktop electron:build
```

Check the build logs for missing signing settings, native dependency rebuild issues, or platform packaging prerequisites.
