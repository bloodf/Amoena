# Search Architecture

## Overview

Amoena provides search across multiple domains: sessions, memory observations, files, and settings. Search is local-first with optional semantic enhancement.

## Search Domains

| Domain | Index | Query Type | Phase |
|--------|-------|-----------|-------|
| Sessions | SQLite FTS5 | Full-text on messages and metadata | MVP |
| Memory | SQLite FTS5 + SQLite-vec | Keyword + semantic (hybrid) | MVP (FTS5), V1.0 (vector) |
| Files | File system + git | Path matching, content grep | MVP |
| Settings | In-memory | Key prefix matching | MVP |
| Commands | In-memory (cmdk) | Fuzzy match | MVP |

## Global Search (Cmd+K)

The command palette (powered by `cmdk`) provides unified search across all domains:

1. User types a query.
2. Results are fetched in parallel from all active domains.
3. Results are ranked by domain priority and relevance.
4. User selects a result to navigate to it.

### Result Ranking

Results are ordered by:
1. **Domain priority:** Commands > Sessions > Files > Memory
2. **Recency:** Within each domain, recent items rank higher
3. **Relevance:** FTS5 rank score for text matches

## Semantic Search (V1.0+)

Memory observations support semantic search via SQLite-vec:
- Embeddings generated by `nomic-embed-text` (local) or `text-embedding-3-small` (cloud)
- Hybrid ranking via Reciprocal Rank Fusion (see `memory-system.md`)
- Semantic search is opt-in and requires embedding model availability

## Performance Targets

| Operation | Target | Notes |
|-----------|--------|-------|
| Command palette open | < 50ms | Cached command list |
| Keyword search (10K sessions) | < 100ms | FTS5 indexed |
| Semantic search (100K observations) | < 200ms | SQLite-vec ANN |
| File path search | < 50ms | Glob matching |
