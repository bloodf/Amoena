# Release Process

## Scope

This document describes the documentation and process side of Amoena releases.

## Before Release

- architecture changes reconciled
- reference docs updated
- prompt catalog updated if implementation sequencing changed
- product/developer docs updated for user-visible features

## Release Artifacts

- desktop release notes
- docs site update
- changelog entry
- compatibility notes for plugins/adapters where needed

## Breaking Changes

When contracts change:

- update architecture docs
- update reference docs
- update prompts
- call out migration expectations
