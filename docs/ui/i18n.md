# Internationalization (i18n)

Lunaria supports 5 languages out of the box using `i18next` and `react-i18next`.

## Supported Languages

| Code | Language |
|------|----------|
| `en` | English |
| `pt-BR` | Portuguese (Brazil) |
| `es` | Spanish |
| `fr` | French |
| `de` | German |

## Package Structure

```
packages/i18n/
├── src/
│   ├── index.ts          # Main exports
│   ├── create-i18n.ts    # i18next initialization
│   ├── react.tsx          # React integration (useTranslation)
│   ├── native.ts          # React Native (Expo) integration
│   ├── types.ts           # Type definitions
│   └── resources/
│       ├── en.ts          # English translations
│       ├── pt-BR.ts       # Portuguese translations
│       ├── es.ts          # Spanish translations
│       ├── fr.ts          # French translations
│       └── de.ts          # German translations
└── package.json
```

## Using Translations

### In React Components

```tsx
import { useTranslation } from "react-i18next";

export function MyComponent() {
  const { t } = useTranslation();

  return (
    <div>
      <h1>{t("myNamespace.title")}</h1>
      <p>{t("myNamespace.description")}</p>
    </div>
  );
}
```

### With Interpolation

```tsx
// en.ts: { counter: { items: "{{count}} items" } }
<span>{t("counter.items", { count: 42 })}</span>
```

## Adding New Translation Keys

### Step 1: Add to English locale

```typescript
// packages/i18n/src/resources/en.ts
export const en = {
  // ... existing keys
  myFeature: {
    title: "My Feature",
    description: "This is my feature",
    action: "Do Something",
    empty: "No items yet",
  },
} as const;
```

### Step 2: Add to all other locale files

Add the same keys to `pt-BR.ts`, `es.ts`, `fr.ts`, and `de.ts`. Use English as placeholder until translations are available.

### Step 3: Use in component

```tsx
const { t } = useTranslation();
// t("myFeature.title") → "My Feature"
```

## i18n Rules

1. **Never hardcode English strings** in UI components
2. **Always use `t()` function** for user-visible text
3. **Add keys to ALL locale files** when adding new strings
4. **Use namespaced keys** (e.g., `permission.title`, not just `title`)
5. **PR checklist**: Verify i18n keys added for any new UI strings

## React Native (Expo)

For the mobile app, use the native integration:

```typescript
import { initI18n } from '@lunaria/i18n/native';

// Uses expo-localization to detect device language
initI18n();
```
