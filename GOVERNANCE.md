# Governance

## Project Structure

Lunaria is maintained by a small core team with an open contribution model.

## Decision Making

- **Architecture decisions** are documented in `docs/architecture/` and require core team review.
- **Feature proposals** are discussed in GitHub Issues before implementation.
- **Bug fixes and small improvements** can be submitted directly as pull requests.

## Roles

| Role | Responsibilities |
|------|-----------------|
| Maintainer | Merge PRs, cut releases, architecture decisions |
| Contributor | Submit PRs, report issues, review code |
| User | Report issues, request features |

## Release Process

See `docs/developer/release-process.md` for the release workflow.

## Code of Conduct

All participants must follow the [Code of Conduct](./CODE_OF_CONDUCT.md).
