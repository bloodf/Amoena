# Color System

Lunaria uses an **HSL-based CSS custom property** system for all colors. Each token is defined as raw HSL values (without the `hsl()` wrapper), which allows easy opacity manipulation via Tailwind's slash syntax.

## How It Works

CSS variables are defined as space-separated HSL values:

```css
:root {
  --primary: 300 100% 36%;
}
```

Consumed through Tailwind config as:

```css
color: hsl(var(--primary));
/* With opacity */
background: hsl(var(--primary) / 0.5);
```

In Tailwind classes: `bg-primary`, `text-primary-foreground`, `border-border`, etc.

---

## Theming

The design system defaults to **dark mode** (`:root`). Light mode activates via the `.light` class on a parent element.

```html
<!-- Dark (default) -->
<div>...</div>

<!-- Light -->
<div class="light">...</div>
```

## Semantic Token Maps (from @lunaria/tokens)

The `@lunaria/tokens` package also exports higher-level semantic color maps for specific UI contexts:

- **`providerIdentityTokens`** — Maps AI providers (claude, opencode, codex, gemini, anthropic, openai, google, ollama) to brand colors.
- **`stateTokens`** — Maps agent states (streaming, thinking, reasoning, active, idle, blocked, etc.) to status colors.
- **`rateLimitPressureTokens`** — Maps rate limit levels (safe, caution, warning, exhausted) to urgency colors.
- **`permissionStateTokens`** — Maps permission states (ask, acceptEdits, fullAccess, denied, pending) to indicator colors.

These are consumed by composite components directly via `@lunaria/tokens`.
