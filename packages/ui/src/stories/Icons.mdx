# Icons

Lunaria uses **Lucide React** (`lucide-react@^0.462.0`) as its icon system. Lucide provides a comprehensive set of clean, consistent SVG icons designed for UI work.

## Import Pattern

```tsx
import { Search, Plus, X, Check } from "lucide-react";
```

Icons render as inline SVGs and accept standard SVG props:

```tsx
<Search className="h-4 w-4" />
<Plus className="h-5 w-5 text-primary" />
<X className="h-4 w-4 text-muted-foreground" />
```

---

## Sizing Convention

Icons in Lunaria typically use these Tailwind size classes:

| Size | Class | Pixels | Usage |
|---|---|---|---|
| Small | `h-3 w-3` | 12×12 | Inline indicators, badges |
| Default | `h-4 w-4` | 16×16 | Buttons, inputs, menu items |
| Medium | `h-5 w-5` | 20×20 | Standalone actions, nav items |
| Large | `h-6 w-6` | 24×24 | Feature icons, empty states |
| XL | `h-8 w-8` | 32×32 | Hero sections, onboarding |

## Usage with Components

Icons are commonly paired with buttons, inputs, and badges:

```tsx
import { Button } from "@lunaria/ui/primitives";
import { Plus, Search, Loader2 } from "lucide-react";

{/* Icon button */}
<Button size="icon" variant="ghost">
  <Plus className="h-4 w-4" />
</Button>

{/* Button with icon + label */}
<Button>
  <Plus className="h-4 w-4 mr-2" />
  New Session
</Button>

{/* Loading state */}
<Button disabled>
  <Loader2 className="h-4 w-4 mr-2 animate-spin" />
  Loading...
</Button>
```

## Icon Aliases

Some components alias icons to avoid naming collisions:

```tsx
import { Monitor as MonitorIcon } from "lucide-react";
```

This pattern is used when a component name might clash with the icon name.
